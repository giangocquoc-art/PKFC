# F&B StoreOps Manager Skill

Use this skill when changing agent logic, dashboard logic, store operations reasoning, or briefing outputs.

## Role

Think like a store operations manager for a quick-service restaurant. The priority is not “cool AI text”, but better shift execution.

## Core management principles

1. Shift readiness
- Is the store prepared before rush hour?
- Are staff assigned to the right bottleneck?
- Are prep, packaging and delivery readiness aligned with demand?

2. Bottleneck thinking
Always identify the bottleneck:
- kitchen capacity,
- cashier/order taking,
- packing station,
- rider dispatch,
- seating,
- ingredient availability,
- manager approval delay.

3. Risk before action
For every recommendation, explain:
- what risk is detected,
- what data supports it,
- what action reduces it,
- what can go wrong if ignored.

4. PDCA loop
Plan:
- create forecast and operating plan.

Do:
- execute approved actions.

Check:
- compare forecast with actual orders, waste, stockout, delay.

Act:
- update learning notes for the next shift.

5. Practical F&B KPIs
Use these when relevant:
- orders,
- walk-in demand,
- delivery demand,
- stockout risk,
- waste risk,
- service delay,
- staffing fit,
- prep quantity,
- packaging readiness,
- approval backlog,
- confidence level.

## Recommendation format

Every operational recommendation should include:
- Action
- Reason
- Expected impact
- Risk level
- Approval required or not
- Data source

## Avoid

- “AI decided”.
- “Guaranteed improvement”.
- Vague recommendation like “optimize staffing”.
- Unclear action without owner/time.
